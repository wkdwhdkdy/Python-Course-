## 개요

Dos attack에는 Syn-flooding Attack 과 같이 다수의 컴퓨터를 이용하여 Victim에게 많은 양의 데이터로 flooding을 일으키는 DDos attack 이 대부분인 줄 알았지만, 전자와 달리 대역폭이 거의 필요가 없고, 한 대의 컴퓨터 또는 한 개의 Attack으로도 사이버 위협을 일으킬 수 있는  Slow Dos Attack이 있다. Slow Dos attack을 Kali Linux를 사용해 공격해보고 방어할 수 있는 방법들에 대하여 알아보았다. 



### Slow HTTP Header Attack

이 공격은 Apache HTTP 서버만 영향을 받는다. Request Header가 도착할 때 까지 connetion을 유지하는 것을 이용한 공격으로  Request Header를 크고 천천히 보내서 connection을 가능한 오래 유지함으로써 서버의 Resource를 고갈시킨다. 또한 HTTP GET 메소드를 사용하여 헤더의 끝을 알리는 개행 문자열 **/r/n/r/n (HEX : 0d 0a 0d 0a)** 을 포함하지 않고 전송하여, Connection을 유지할 수도 있다. 
**Apache 2.4**에서는 **LimitedRequestFieldSize**, **LimitedRequestFields**를 통하여 Header의 사이즈를 제한한다. 각각의 기본값은 8190bytes와 100개 이다. 즉 공격자는 819KB로 100개의 header를 보낼 수 있다.[1] 

Slow HTTP Header공격은 Bee-box와 Kali Linux의 metasploit을 이용한다. 

VirtualBox를 사용해서 Bee-Box와 Kali Linux 를 설치하고, Kali linux의 Metasploit 내부에 있는 **Slowloris Tool**을 사용하여 Bee-box에서 제공하는 Apache HTTP server를 공격한다. 

공격하기에 앞서 정상적인 Request Header는 그림 1과 같이 **/r/n/r/n (HEX : 0d 0a 0d 0a)** 으로 끝나는 것을 확인 할 수 있다.

### Slow HTTP POST Attack

 Slow HTTP Header Attack 과 비슷하게 큰 Request Body를 전송하는 동안 공격자가 서버와 지속적인 연결을 할 수있다.  Apache 2.4에서는 **LimitedRequestBody로** request body의 크기를 정할 수 있다. 범위는 0B부터 2GB 까지이며, Default 값은 0(meaning unlimited)이다[2]. 즉, HTTP의 POST 메소드를 사용하여 헤더의 Content-length 값을 큰 값으로 조작하여 전송함으로써 웹 서버와의 Connection 을 최대한 오랫동안 유지하여 서버의 Resource를 고갈시키는 공격이다. 



그림 5와 같이 Kali Linux에서 Slowhttptest를 사용해서 Apache HTTP server 에 공격을 실행한다.

### Slow Read Attack 

공격자가 TCP Segment Header 중 **TCP Window Size** 를 0 또는 작은 크기로 조작하여 서버로부터 천천히 응답을 받아 연결을 유지한다. 해당 공격 기법의 대부분은 Apache HTTP Server가 연결 지연을 제한하지 않는 사실을 이용한다. TCP Window Size를 공격자가 임의로 0이상의 값으로 정할 수 있다.  TCP Window Size 작을수록 공격자가 연결을 유지하기가 더 쉽지만 서버에게 공격을 감지당하기도 쉬워진다. 

**Slowhttptest**의 옵션 중 -w, -y를 사용해서 TCP Window Size의 범위를 지정해줄 수 있다. 기본값은 1~512이다.

## 방어 기법[3]

### Increasing the limit of simultaneous requests 

​	 동시에 발생하는 request의 처리 횟수를 증가시켜서 connection pool의 자원 고갈을 방지한다.  Apache 2.4에서는 MaxRequesetWorkers 로 정할 수 있다. 기본값은 150이다.[2]

### Shortening the time out value 

​	Apache 2.4에서 Timeout 값을 짧게 만들어서 Slow Dos Attack 이 들어와도 연결을 오랫동안 유지할 수 없게 한다. 하지만 Timeout 값이 짧아지면 서비스의 품질이 떨어지기 때문에 Timeout 값을 줄이는 데 한계가 존재한다.

### Limiting the number of requests for each client

​	각각의 사용자가 쓸 수 있는 request의  개수를 정하여 공격자가 한 명인 경우에는 공격이 성공하지 못하게 하는 방법이다. 

[1]에서는 Slow Dos Attack을 할 때 필요한 Cost 값을 증가시켜,  공격자가 공격을 하기 위해서 많은 Resource를 사용하는 방법을 제시하였다. 



## 결론

​	실습을 하면서 각각의 공격을 해보니,  본문에서 나온 **LimitedRequestFieldSize**, **LimitedRequestFields*****LimitedRequestBody**, **LimitedRequestFieldSize**, **LimitedRequestFields** 와 같은 directive 들을 설정해주면 Slow Dos Attack을 방어하기에 한층 수월할 것 같다. 본문에서 패킷을 확인했을 때 정상적인 패킷과 다르게 TCP Window Size, Content-length에서 차이점이 존재한다.  게다가 Apache HTTP Server에 행해지는 공격들의 대부분은 암호화된 트래픽이 존재하지 않는다.  그러한 트래픽들을 모아 차이점을 토대로 시그니처 기반의 탐지를 한다면, Slow Dos Attack을 탐지할 수 있지 않을까라는 생각이 든다.
